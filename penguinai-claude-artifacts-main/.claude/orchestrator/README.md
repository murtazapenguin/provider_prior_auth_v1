# Orchestrator Guide

Phase-by-phase instructions for the orchestrator. Read ONLY the file for your current phase.

| Current Phase | Read | Sections Covered |
|---------------|------|-----------------|
| Phase 0 (requirements) | `requirements.md` | Capability selection, env vars, contracts, plan mode, checklist, test fixtures |
| Phase 0.5 (infrastructure) | `infrastructure.md` | Service startup, .env files, S3 validation |
| Spawning + verifying agents | `spawning.md` | Phase instructions, plan mode protocol, verify→resume, parallel phases |
| Phase 0 design reference | `templates.md` | Data model, workflow state machine, processing status, API shapes |
| After Phase 3 (wrap-up) | `completion.md` | Final output format, post-task cleanup |

For the overall pipeline overview, see `CLAUDE.md` Section 8 (Phase Workflow).
